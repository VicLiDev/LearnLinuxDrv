## 6. Target 目标系统

### 6.1 核心数据结构

```c
enum target_state {
    TARGET_UNKNOWN = 0,        // 未知状态
    TARGET_RUNNING = 1,        // 正在执行用户代码
    TARGET_HALTED = 2,         // 调试模式，已暂停
```

### 6.2 target_type 接口 (虚函数表)

`src/target/target_type.h` 定义了所有目标类型必须实现的接口：

```c
struct target_type {
    const char *name;               // 类型名

    // 生命周期
    int (*init_target)(struct command_context *cmd_ctx, struct target *target);
    int (*examine)(struct target *target);           // 探测目标
    int (*poll)(struct target *target);              // 轮询目标状态

    // 执行控制
    int (*halt)(struct target *target);              // 暂停
    int (*resume)(struct target *target, bool current,
                  target_addr_t address, bool handle_breakpoints,
                  bool debug_execution);
    int (*step)(struct target *target, bool current,
                target_addr_t address, bool handle_breakpoints);

    // 复位
    int (*assert_reset)(struct target *target);
    int (*deassert_reset)(struct target *target);
    int (*soft_reset_halt)(struct target *target);

    // 寄存器访问
    int (*get_gdb_reg_list)(struct target *target, struct reg **reg_list[],
                           int *reg_list_size,
                           enum target_register_class reg_class);

    // 内存访问
    int (*read_memory)(struct target *target, target_addr_t address,
                      uint32_t size, uint32_t count, uint8_t *buffer);
    int (*write_memory)(struct target *target, target_addr_t address,
                       uint32_t size, uint32_t count, const uint8_t *buffer);

    // 断点/观察点
    int (*add_breakpoint)(struct target *target, struct breakpoint *breakpoint);
    int (*remove_breakpoint)(struct target *target, struct breakpoint *breakpoint);
    int (*add_watchpoint)(struct target *target, struct watchpoint *watchpoint);
    int (*remove_watchpoint)(struct target *target, struct watchpoint *watchpoint);

    // 算法执行 (Flash 烧写等)
    int (*run_algorithm)(struct target *target, int num_mem_params,
                        struct mem_param *mem_params, int num_reg_params,
                        struct reg_param *reg_params,
                        target_addr_t entry_point,
                        target_addr_t exit_point, int timeout_ms, void *arch_info);

    // GDB 集成
    const char *(*get_gdb_arch)(const struct target *target);
    // ... 更多方法
};
```

### 6.4 断点系统

**断点结构** (`src/target/breakpoints.c`)：
```c
struct breakpoint {
    target_addr_t address;      // 断点地址
    uint32_t asid;              // 地址空间 ID
    unsigned int length;        // 断点长度
    enum breakpoint_type type;  // 硬件/软件断点
    bool is_set;               // 是否已设置
    uint8_t *orig_instr;       // 被替换的原始指令 (软件断点)
    struct breakpoint *next;   // 链表
};
```

**断点类型**：
- **软件断点** (`BKPT_SOFT`)：替换目标地址的指令为 BKPT 指令，触发调试异常
- **硬件断点** (`BKPT_HARD`)：使用目标芯片内部的断点比较单元，不修改代码

**断点操作流程**：
```
breakpoint_add()
  → breakpoint_add_internal()
    → target_add_breakpoint()
      → target->type->add_breakpoint()
        → cortex_m_add_breakpoint() / cortex_a_add_breakpoint() / ...
```

### 6.5 事件系统

Target 系统提供丰富的事件回调机制：

```c
// 关键事件
TARGET_EVENT_HALTED          // 目标暂停
TARGET_EVENT_RESUMED         // 目标恢复运行
TARGET_EVENT_RESET_START     // 复位开始
TARGET_EVENT_RESET_INIT      // 复位后 init
TARGET_EVENT_RESET_DEASSERT  // 复位释放
TARGET_EVENT_EXAMINE_START   // 探测开始
TARGET_EVENT_EXAMINE_FAIL    // 探测失败
TARGET_EVENT_GDB_HALT        // GDB 暂停通知
TARGET_EVENT_DEBUG_RESUMED   // 调试运行恢复

// 注册回调
target_register_event_callback(callback_func, private_data);
```
