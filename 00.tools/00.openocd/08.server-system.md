## 8. Server 服务端系统

### 8.1 服务器框架

**服务端核心结构** (`src/server/server.h`)：

```c
// 连接抽象
struct connection {
    int fd;                 // 文件描述符
    int fd_out;             // 输出描述符 (管道)
    struct sockaddr_in sin; // 网络地址
    struct command_context *cmd_ctx;  // 命令上下文
    struct service *service;         // 所属服务
    void *priv;            // 服务私有数据
    struct connection *next;  // 连接链表
};

// 服务抽象
struct service {
    char *name;                   // 服务名
    enum connection_type type;    // 连接类型
    char *port;                   // 端口号
    unsigned short portnumber;    // 端口号数值
    int fd;                       // 监听 socket
    void *priv;
};

// 服务驱动接口
struct service_driver {
    const char *name;
    int (*new_connection)(struct connection *connection);
    int (*input)(struct connection *connection);
    int (*connection_closed)(struct connection *connection);
    int (*keep_alive)(void);
};
```

### 8.2 主事件循环

`src/server/server.c:431` 中的 `server_loop()`：

```c
while (shutdown_openocd == CONTINUE_MAIN_LOOP) {
    fd_set read_fds;
    FD_ZERO(&read_fds);

    // 将所有服务的监听 fd 和连接 fd 加入 select
    // ...

    // 使用 select 等待事件
    retval = socket_select(fd_max + 1, &read_fds, NULL, NULL, &tv);

    if (retval == 0) {
        // 超时：执行定时器回调
        target_call_timer_callbacks();
        process_jim_events(command_context);
    } else {
        // 处理新连接和输入数据
        for (each_service)
            if (FD_ISSET(service->fd, &read_fds))
                add_connection(service);

        for (each_connection)
            if (FD_ISSET(connection->fd, &read_fds))
                service->input(connection);
    }
}
```

### 8.3 GDB 服务器

`src/server/gdb_server.c` 实现了 **GDB 远程串行协议 (RSP)**：

```
GDB Client ←→ TCP Socket ←→ gdb_server.c ←→ Target System
```

**GDB RSP 包格式**：
```
发送: $packet-data#checksum
应答: + (ACK) 或 - (NAK)
示例: $m4000,4#9a  (读取地址 0x4000 处的 4 字节)
```

**GDB 连接私有数据**：
```c
struct gdb_connection {
    char buffer[GDB_BUFFER_SIZE + 1];  // 接收缓冲区
    bool ctrl_c;                        // Ctrl-C 中断
    enum target_state frontend_state;   // GDB 看到的目标状态
    struct image *vflash_image;         // 虚拟 Flash 镜像
    bool closed;                        // 连接是否关闭
};
```

**GDB 服务器处理的主要 RSP 命令**：

| 命令              | 说明           |
|-------------------|----------------|
| `m addr,len`      | 读取内存       |
| `M addr,len:data` | 写入内存       |
| `g`               | 读取所有寄存器 |
| `G data`          | 写入所有寄存器 |
| `p n`             | 读取单个寄存器 |
| `P n=value`       | 写入单个寄存器 |
| `c` / `c addr`    | 继续执行       |
| `s` / `s addr`    | 单步执行       |
| `Z0,addr,len`     | 设置软件断点   |
| `Z1,addr,len`     | 设置硬件断点   |
| `z0,addr,len`     | 删除断点       |
| `qSupported`      | 查询支持特性   |
| `vFlashErase`     | Flash 擦除     |
| `vFlashWrite`     | Flash 写入     |
| `vFlashDone`      | Flash 操作完成 |

### 8.4 Telnet 服务器

`src/server/telnet_server.c` 提供交互式命令行接口：

```
telnet localhost 4444
> halt
> resume
> mdw 0x20000000 16
> flash write_image erase test.bin 0x08000000
> reset run
```

### 8.5 RTT 服务器

`src/server/rtt_server.c` 支持 SEGGER RTT (Real-Time Transfer)，可在调试时实时输出目标程序的日志。
