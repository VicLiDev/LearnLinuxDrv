## 9. 配置系统 (TCL)

### 9.1 配置文件体系

OpenOCD 使用 TCL 脚本作为配置语言，典型的配置文件层次：

```
openocd.cfg              # 顶层配置
├── interface/cmsis-dap.cfg    # 调试器配置
├── target/stm32f4x.cfg        # 目标芯片配置
├── flash/stm32f4x.cfg         # Flash 配置 (通常在 target 配置中包含)
└── board/my_board.cfg         # 板级配置
```

### 9.2 常用配置命令

**调试器配置**：
```tcl
# 选择调试器
interface cmsis-dap
adapter speed 4000          # 设置 JTAG/SWD 时钟 (kHz)
transport select swd        # 选择传输协议
```

**JTAG TAP 配置**：
```tcl
jtag newtap chip tap -irlen 4 -expected-id 0x12345678
```

**目标配置**：
```tcl
# 创建目标
set CHIPNAME stm32f407vg
cortex_m create_target $_TARGETNAME
target create $_TARGETNAME cortex_m -chain-position $_CHIPNAME.dap

# 配置
$_TARGETNAME configure -work-area-phys 0x20000000 -work-area-size 0x4000
$_TARGETNAME configure -event examine-start { ... }
```

**Flash 配置**：
```tcl
set FLASH_SIZE 0x100000
flash bank $_FLASHNAME stm32f2x 0x08000000 0 $_FLASH_SIZE 0 $_TARGETNAME
```

**复位配置**：
```tcl
reset_config srst_only srst_push_pull
# 或
reset_config trst_and_srst
$_TARGETNAME configure -event reset-init {
    # 复位后执行的初始化脚本
    ...
}
```

### 9.3 启动内嵌 TCL 脚本

`src/openocd.c:48` 嵌入的启动 TCL 脚本提供了基础命令：

```tcl
# script 命令 - 查找并执行配置文件
proc script {filename} {
    # 在搜索路径中查找文件
    # 找到后 source 执行
}

# add_script_search_dir - 添加搜索路径
# _run_post_init_commands - 执行 post-init 钩子
```

### 9.4 事件钩子系统

TCL 配置中可以注册事件回调：

```tcl
# 在目标暂停时执行
$_TARGETNAME configure -event gdb-attach {
    halt
    reset init
}

# 在复位后执行
$_TARGETNAME configure -event reset-init {
    # 配置时钟、GPIO 等
    mww 0x40023830 0x00000001
}

# 在探测失败时执行
$_TARGETNAME configure -event examine-fail {
    echo "Target detection failed"
}
```
