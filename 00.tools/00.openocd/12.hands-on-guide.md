### 12.1 安装 OpenOCD

**Ubuntu / Debian：**

```bash
# 方式一：包管理器安装（版本可能较旧）
sudo apt install openocd

# 验证安装
openocd --version
openocd -l   # 列出所有已注册的调试器和目标配置
```

**macOS：**

```bash
brew install openocd
```

**Windows：**

从 [OpenOCD 官网](https://github.com/openocd-org/openocd/releases) 下载 zip 包，解压后直接使用 `bin/openocd.exe`。

> 也可以从源码编译以获取最新特性，见 [12.10 节](#1210-从源码编译-openocd)。

### 12.2 连接调试器与目标板

#### 12.2.1 常见调试器硬件

| 调试器            | 协议     | OpenOCD interface 名称        | 备注                      |
|-------------------|----------|-------------------------------|---------------------------|
| ST-Link V2/V3     | SWD/JTAG | `stlink` / `stlink.cfg`       | STM32 常用，性价比高      |
| J-Link            | SWD/JTAG | `jlink` / `jlink.cfg`         | SEGGER 出品，兼容性好     |
| CMSIS-DAP         | SWD/JTAG | `cmsis-dap` / `cmsis-dap.cfg` | 通用协议，多数开发板自带  |
| FT2232            | JTAG/SWD | `ftdi` 系列                   | 基于 FTDI 芯片的 DIY 方案 |
| DAPLink           | SWD/JTAG | `cmsis-dap`                   | ARM 官方开源方案          |

#### 12.2.2 SWD 接线（最常用，2 线）

```
调试器                          目标板
──────                          ──────
SWDIO  ───────────────────────  SWDIO
SWCLK  ───────────────────────  SWCLK
GND    ───────────────────────  GND
VTREF  ───────────────────────  VCC (3.3V，仅检测电压)
```

> **注意**：
> - GND 必须连接，VTREF 用于检测目标板电压，建议连接
> - 不要给调试器单独供电超过目标板电压
> - 如果使用 SWO（串行输出），额外连接 SWO 引脚

#### 12.2.3 JTAG 接线（4 + 2 线）

```
调试器                          目标板
──────                          ──────
TCK    ───────────────────────  TCK
TMS    ───────────────────────  TMS
TDI    ───────────────────────  TDI
TDO    ───────────────────────  TDO
GND    ───────────────────────  GND
VTREF  ───────────────────────  VCC (3.3V)
(可选) SRST ───────────────────  nRST (系统复位)
(可选) TRST ───────────────────  TRST (测试复位)
```

#### 12.2.4 检查调试器是否被识别

```bash
# Linux - 查看是否识别到 USB 设备
lsusb | grep -i -E "stlink|jlink|cmsis|ftdi"

# 查看 udev 规则是否生效（权限问题常见原因）
ls -la /dev/bus/usb/*/*/

# 如果没有权限，添加 udev 规则（以 ST-Link 为例）
# sudo cp /usr/share/openocd/contrib/60-openocd.rules /etc/udev/rules.d/
# sudo udevadm control --reload-rules && sudo udevadm trigger
```

### 12.3 编写第一个配置文件

以 **STM32F103C8T6 (Blue Pill)** + **ST-Link V2** 为例。

创建文件 `stm32f103.cfg`：

```tcl
# ============================================
# STM32F103C8T6 配置文件
# 调试器: ST-Link V2
# ============================================

# ----- 第一步：选择调试器 -----
source [find interface/stlink.cfg]

# 设置 SWD 时钟频率 (kHz)
# 起步建议用较低频率，稳定后再调高
adapter speed 1000

# 选择传输协议
transport select swd

# ----- 第二步：配置目标芯片 -----
source [find target/stm32f1x.cfg]
```

**以 STM32F407 + J-Link 为例：**

```tcl
# J-Link 调试器
source [find interface/jlink.cfg]
adapter speed 4000
transport select swd

# STM32F4 目标
set CHIPNAME stm32f407vg
source [find target/stm32f4x.cfg]
```

**查找内置配置文件：**

```bash
# 查看所有内置的 interface 配置
ls /usr/share/openocd/scripts/interface/

# 查看所有内置的 target 配置
ls /usr/share/openocd/scripts/target/

# 搜索特定芯片的配置
find /usr/share/openocd/scripts/ -name "*stm32f1*"
find /usr/share/openocd/scripts/ -name "*imx6*"
```

> **配置文件查找路径**：OpenOCD 默认在以下位置搜索配置文件：
> - `-s` 参数指定的路径
> - 当前目录
> - 安装目录下的 `share/openocd/scripts/`

### 12.4 启动 OpenOCD 并验证连接

#### 12.4.1 基本启动

```bash
# 使用配置文件启动
openocd -f stm32f103.cfg

# 指定配置文件路径和搜索路径
openocd -s /usr/share/openocd/scripts -f interface/stlink.cfg -f target/stm32f1x.cfg
```

成功启动后你会看到类似输出：

```
Open On-Chip Debugger 0.12.0
Licensed under GNU GPL v2
For bug reports, read http://openocd.org/doc/doxygen/bugs.html
Info : auto-selecting first available session transport "swd". To override use 'transport select <transport>'.
Info : Using ST-Link v2
Info : Target voltage: 3.250V
Info : stm32f1x.cpu: hardware has 6 breakpoints, 4 watchpoints
Info : starting gdb server for stm32f1x.cpu on 3333
Info : Listening on port 3333 for gdb connections
Info : accepting 'telnet' connection on tcp/4444
```

**关键信息解读：**

| 输出信息                                    | 含义                           |
|---------------------------------------------|--------------------------------|
| `Target voltage: 3.250V`                    | 检测到目标板电压，确认接线正确 |
| `hardware has 6 breakpoints, 4 watchpoints` | 芯片硬件调试资源               |
| `gdb server ... on 3333`                    | GDB 远程调试端口               |
| `telnet ... on 4444`                        | Telnet 命令行端口              |

#### 12.4.2 命令行快速验证

```bash
# 启动时直接执行命令，验证后自动退出
openocd -f stm32f103.cfg -c "init" -c "targets" -c "exit"
```

输出示例：

```
    TargetName         Type       Endian TapName            State
--  ------------------ ---------- ------ ------------------  --------
 0* stm32f1x.cpu       cortex_m   little stm32f1x.dap       reset
```

> `State` 为 `reset` 表示连接成功，目标芯片已被识别。

#### 12.4.3 常用启动参数

```
openocd [选项]
  -f <file>       加载配置文件（可多次使用）
  -s <dir>        添加配置文件搜索路径
  -c <command>    执行一条 TCL 命令（可多次使用）
  -d <level>      设置调试日志级别 (0-4)
  -l <file>       将日志输出到文件
  --search        显示默认搜索路径并退出
  -l, --list      列出所有已注册的调试器和目标
```

### 12.5 Flash 烧写实操

#### 12.5.1 查看当前 Flash 信息

```bash
# 启动 OpenOCD 后，另开终端连接 Telnet
telnet localhost 4444

# 在 Telnet 会话中执行：
flash probe 0
flash info 0
```

输出示例：

```
# Flash bank 0 信息
flash bank 0: stm32f1x (aspect ratio: x8, size: 64kB, base: 0x08000000)
  #0: 0x08000000 (16kB) - 保护: 关
  #1: 0x08004000 (16kB) - 保护: 关
  #2: 0x08008000 (16kB) - 保护: 关
  #3: 0x0800c000 (16kB) - 保护: 关
```

#### 12.5.2 擦除 Flash

```tcl
# 擦除整个 Flash
flash erase_sector 0 0 last

# 擦除指定扇区（扇区 0 和 1）
flash erase_sector 0 0 1

# 擦除整个芯片
flash erase_bank 0
```

#### 12.5.3 烧写固件

```tcl
# 烧写 .bin 文件（需要指定地址）
flash write_image erase build/my_firmware.bin 0x08000000 bin

# 烧写 .hex 文件（地址由 HEX 文件指定）
flash write_image erase build/my_firmware.hex

# 烧写 .elf 文件（推荐，包含完整的地址和符号信息）
flash write_image erase build/my_firmware.elf

# 仅写入，不擦除（需要提前手动擦除）
flash write_image build/my_firmware.bin 0x08000000 bin

# 写入并验证
flash write_image erase verify build/my_firmware.elf
```

> **参数说明**：
> - `erase`：写入前自动擦除（只擦除需要的扇区）
> - `unlock`：写入前自动解锁（如果 Flash 被锁定）
> - `verify`：写入后自动校验

#### 12.5.4 一条命令完成烧写并复位运行

```bash
openocd -f stm32f103.cfg \
  -c "init" \
  -c "reset halt" \
  -c "flash write_image erase build/my_firmware.elf" \
  -c "verify_image build/my_firmware.elf" \
  -c "reset run" \
  -c "shutdown"
```

> 这是最常用的命令行烧写方式，适合集成到 Makefile 或 IDE 中。

#### 12.5.5 读取 Flash 内容

```tcl
# 将 Flash 内容导出为二进制文件
flash read_bank 0 flash_dump.bin

# 读取指定范围
flash read_bank 0 flash_dump.bin 0x08000000 0x1000
```

#### 12.5.6 在 Makefile 中集成烧写

```makefile
# Makefile 片段
OPENOCD = openocd -f stm32f103.cfg

flash:
	$(OPENOCD) -c "init" -c "reset halt" \
		-c "flash write_image erase $(BUILD_DIR)/$(TARGET).elf" \
		-c "verify_image $(BUILD_DIR)/$(TARGET).elf" \
		-c "reset run" -c "shutdown"

erase:
	$(OPENOCD) -c "init" -c "flash erase_bank 0" -c "shutdown"

debug:
	$(OPENOCD) -f stm32f103.cfg
```

### 12.6 搭建 GDB 远程调试环境

#### 12.6.1 整体架构

```
┌─────────┐    gdb remote     ┌───────────┐    SWD/JTAG     ┌──────────┐
│  GDB    │ ────────────────> │ OpenOCD   │ ──────────────> │ 目标板   │
│ (arm-   │    port 3333      │ (调试代理)│                 │ (STM32)  │
│  none-  │ <──────────────── │           │ <────────────── │          │
│  eabi-  │                   │           │                 │          │
│  gdb)   │                   └───────────┘                 └──────────┘
└─────────┘
```

#### 12.6.2 第一步：安装交叉编译 GDB

```bash
# Ubuntu - 安装 ARM 交叉编译工具链
sudo apt install gcc-arm-none-eabi
sudo apt install gdb-multiarch    # 通用版，支持多架构

# 或者安装专用的 ARM GDB
sudo apt install gdb-arm-none-eabi
```

#### 12.6.3 第二步：启动 OpenOCD（在一个终端）

```bash
# 启动 OpenOCD，保持运行
openocd -f stm32f103.cfg
```

#### 12.6.4 第三步：连接 GDB（在另一个终端）

```bash
# 使用带调试信息的 ELF 文件启动 GDB
arm-none-eabi-gdb build/my_firmware.elf
```

#### 12.6.5 GDB 常用调试命令

```gdb
# ----- 连接 -----
(gdb) target remote :3333            # 连接 OpenOCD 的 GDB 服务

# ----- 复位与运行 -----
(gdb) monitor reset halt             # 复位并暂停（monitor 前缀 = 发给 OpenOCD 的命令）
(gdb) monitor reset run              # 复位并运行
(gdb) load                           # 烧写当前 ELF 到 Flash
(gdb) continue / c                   # 继续运行

# ----- 断点 -----
(gdb) break main                     # 在 main 函数设断点
(gdb) break file.c:42                # 在 file.c 第 42 行设断点
(gdb) break *0x08000123              # 在指定地址设断点
(gdb) info breakpoints                # 查看所有断点
(gdb) delete 1                       # 删除断点 1

# ----- 单步执行 -----
(gdb) step / s                       # 单步进入（会进入函数内部）
(gdb) next / n                       # 单步跳过（不进入函数内部）
(gdb) stepi / si                     # 指令级单步进入
(gdb) nexti / ni                     # 指令级单步跳过
(gdb) finish                        # 运行到当前函数返回

# ----- 查看变量和内存 -----
(gdb) print var                      # 打印变量值
(gdb) print /x var                   # 以十六进制打印
(gdb) print &var                     # 打印变量地址
(gdb) x/16xb 0x20000000              # 查看 16 字节内存（十六进制，按字节）
(gdb) x/4xw 0x20000000              # 查看 4 个字（32 位，十六进制）
(gdb) display var                    # 每次暂停自动打印变量
(gdb) info registers                 # 查看所有寄存器
(gdb) info registers r0 r1 sp       # 查看指定寄存器

# ----- 栈与调用 -----
(gdb) backtrace / bt                 # 打印调用栈
(gdb) frame 2                       # 切换到栈帧 2
(gdb) info frame                     # 查看当前栈帧信息

# ----- 线程/任务 -----
(gdb) info threads                   # 查看所有线程
(gdb) thread 2                      # 切换到线程 2

# ----- 杂项 -----
(gdb) set var counter=100            # 修改变量值
(gdb) monitor reset halt             # monitor = 发送命令给 OpenOCD
(gdb) quit                           # 退出 GDB
```

#### 12.6.6 典型调试流程

```gdb
$ arm-none-eabi-gdb build/my_firmware.elf

# 1. 连接
(gdb) target remote :3333
# 0x08000000 in ?? ()

# 2. 复位到起始位置

# 3. 烧写程序

Loading section .rodata, size 0x100 lma 0x80023c0
...
Start address 0x8000124, load size 0x3a64

# 4. 在 main 设断点
(gdb) break main
Breakpoint 1 at 0x8000234: file src/main.c, line 15.

# 5. 运行到断点
(gdb) continue
Breakpoint 1, main () at src/main.c:15
15      int counter = 0;

# 6. 单步调试
(gdb) next
16      HAL_Init();
(gdb) step
```

### 12.7 常用调试操作

#### 12.7.1 内存读写

```tcl
# 读取内存 (Telnet 会话中)
mdw 0x20000000 8          # 读取 8 个 32 位字
mdh 0x20000000 8          # 读取 8 个 16 位半字
mdb 0x20000000 16         # 读取 16 个字节

# 写入内存
mww 0x20000000 0x12345678 # 写入 32 位
mwh 0x20000000 0x1234     # 写入 16 位
mwb 0x20000000 0x12       # 写入 8 位

# 在 GDB 中
(gdb) x/8xw 0x20000000
(gdb) set {unsigned int}0x20000000 = 0x12345678
```

#### 12.7.2 寄存器读写

```tcl
# 读取所有寄存器
reg

# 读取指定寄存器
reg r0
reg sp
reg pc
reg xPSR

# 写入寄存器
reg r0 0x12345678
reg pc 0x08000000
```

#### 12.7.3 复位控制

```tcl
# 复位并暂停（调试用）
reset halt

# 复位并运行
reset run

# 复位并初始化（执行 reset-init 事件）
reset init

# 查看当前复位配置
reset_config
```

#### 12.7.4 半主机 (Semihosting) 配置

半主机允许目标程序通过调试通道执行 I/O 操作（如 printf 输出到主机）。

```tcl
# 启用半主机
arm semihosting enable

# 在 GDB 中也可以通过 monitor 命令设置
(gdb) monitor arm semihosting enable
```

#### 12.7.5 SWO / ITM 跟踪输出

```tcl
# 配置 SWO 跟踪（在配置文件或 Telnet 中）
tpiu configure external stm32f1x.swo freq 2000000 uart off 5000000

# 或在配置文件中
trace enable
itm port 0 on
```

### 12.8 Telnet 交互与脚本自动化

#### 12.8.1 连接 Telnet

```bash
# OpenOCD 启动后默认监听 4444 端口
telnet localhost 4444

# 或使用自定义端口（在配置文件中设置）
# telnet_interface port 5555
```

#### 12.8.2 常用 Telnet 命令速查

```tcl
# 目标状态
targets                           # 列出所有目标
reg                               # 显示寄存器
halt                              # 暂停目标
resume                            # 恢复运行
step                              # 单步执行

# Flash 操作
flash probe 0                     # 探测 Flash
flash info 0                      # 查看 Flash 信息
flash erase_sector 0 0 last       # 擦除全部
flash write_image erase fw.elf    # 烧写固件

# 调试器状态
adapter speed 4000                # 设置时钟频率
adapter speed                     # 查看当前频率

# 退出
shutdown                          # 关闭 OpenOCD
exit                              # 断开 Telnet 连接
```

#### 12.8.3 TCL 脚本自动化

创建 `flash_and_run.tcl`：

```tcl
# flash_and_run.tcl - 自动烧写并运行脚本
# 使用方式: openocd -f stm32f103.cfg -f flash_and_run.tcl

# 定义变量
set FIRMWARE_PATH "build/my_firmware.elf"

# 初始化
init

# 复位暂停
reset halt

# 烧写
echo "正在烧写 $FIRMWARE_PATH ..."
flash write_image erase $FIRMWARE_PATH

# 校验
echo "正在校验 ..."
verify_image $FIRMWARE_PATH

# 复位运行
echo "烧写完成，正在复位运行 ..."
reset run

# 关闭
shutdown
```

运行脚本：

```bash
openocd -f stm32f103.cfg -f flash_and_run.tcl
```

#### 12.8.4 在配置文件中定义自定义命令

```tcl
# 在 .cfg 文件中定义
proc my_flash {elf_file} {
    reset halt
    flash write_image erase $elf_file
    verify_image $elf_file
    reset run
    echo "Flash 成功: $elf_file"
}

# 使用：在 Telnet 中执行
# my_flash build/my_firmware.elf
```

### 12.9 常见问题排查

#### 12.9.1 无法识别调试器

**症状**：`Error: libusb_open() failed with LIBUSB_ERROR_ACCESS`

**解决**：

```bash
# 检查设备是否被识别
lsusb

# 检查权限
ls -la /dev/bus/usb/XXX/XXX

# 方案一：添加 udev 规则
sudo cp /usr/share/openocd/contrib/60-openocd.rules /etc/udev/rules.d/
sudo udevadm control --reload-rules
sudo udevadm trigger

# 方案二：临时使用 sudo（不推荐）
sudo openocd -f stm32f103.cfg

# 方案三：将用户添加到 plugdev 组
sudo usermod -aG plugdev $USER
# 重新登录后生效
```

#### 12.9.2 连接超时或失败

**症状**：`Error: Could not find ST-Link device` 或 `Error: target not found`

**检查清单**：

```tcl
# 1. 检查接线（最常见原因）
#    - GND 是否连接
#    - SWDIO / SWCLK 是否接反
#    - 线缆是否松动

# 2. 降低时钟频率
adapter speed 100       # 降到最低

# 3. 检查目标板供电
#    - 目标板是否正常上电
#    - 电压是否在 1.8V ~ 3.3V 范围

# 4. 检查是否有其他程序占用调试器
# Windows: 设备管理器中检查
# Linux: lsof | grep -i usb
```

#### 12.9.3 Flash 解锁

**症状**：`Error: flash bank ... is protected` 或 `Error: target not halted`

```tcl
# 解锁 STM32 Flash
stm32f1x unlock 0
# 然后擦除
flash erase_sector 0 0 last
# 然后重新锁定（可选）
stm32f1x lock 0
```

#### 12.9.4 GDB 连接失败

**症状**：`localhost:3333: Connection refused`

```bash
# 1. 确认 OpenOCD 正在运行
ps aux | grep openocd

# 2. 确认端口正确
netstat -tlnp | grep 3333

# 3. 检查防火墙
sudo ufw allow 3333/tcp

# 4. 如果端口被占用，在配置文件中修改
# gdb_port 4444
```

### 12.10 从源码编译 OpenOCD

#### 12.10.1 依赖安装

```bash
# Ubuntu/Debian
sudo apt install build-essential pkg-config git autoconf \
    automake libtool texinfo libusb-1.0-0-dev libusb-compat-0.1-dev \
    libhidapi-dev libftdi1-dev libjaylink-dev

# 如需 Capstone 反汇编支持
sudo apt install libcapstone-dev

# 如需 Enchant 拼写检查（Jim TCL）
sudo apt install libenchant-2-dev
```

#### 12.10.2 编译步骤

```bash
# 克隆源码
git clone https://github.com/openocd-org/openocd.git
cd openocd

# 切换到稳定版本（推荐）
git checkout v0.12.0    # 替换为最新稳定版本

# 生成配置脚本
./bootstrap

# 配置 - 启用需要的调试器和功能
./configure \
    --enable-stlink \
    --enable-jlink \
    --enable-cmsis-dap \
    --enable-ftdi \
    --enable-jtag_vpi \
    --enable-buspirate \
    --enable-sysfsgpio \
    --enable-linuxgpiod \
    --enable-cmsis-dap-v2

# 编译（使用多线程加速）
make -j$(nproc)

# 安装（可选，不安装也可以直接使用 src/openocd）
sudo make install

# 验证
./src/openocd --version
```

#### 12.10.3 仅编译特定调试器

如果只需要某几个调试器支持，可以减小编译体积：

```bash
./configure --enable-stlink --enable-cmsis-dap --disable-jlink
make -j$(nproc)
```

#### 12.10.4 开发调试

```bash
# 使用 gdb 调试 OpenOCD 本身
gdb --args ./src/openocd -f stm32f103.cfg -d3

# 使用 AddressSanitizer 检测内存问题（开发用）
./configure CFLAGS="-g -fsanitize=address" LDFLAGS="-fsanitize=address"
make -j$(nproc)
```

### 总结：日常使用速查表

| 操作         | 命令                                                                                                            |
|--------------|-----------------------------------------------------------------------------------------------------------------|
| 烧写固件     | `openocd -f xxx.cfg -c "init" -c "reset halt" -c "flash write_image erase fw.elf" -c "reset run" -c "shutdown"` |
| 启动调试代理 | `openocd -f xxx.cfg`                                                                                            |
| GDB 连接     | `arm-none-eabi-gdb fw.elf` → `target remote :3333`                                                              |
| 烧写并运行   | `(gdb) load` → `(gdb) continue`                                                                                 |
| 查看寄存器   | Telnet 中: `reg`，GDB 中: `info registers`                                                                      |
| 读内存       | Telnet: `mdw 0x20000000 8`，GDB: `x/8xw 0x20000000`                                                             |
| 写内存       | `mww 0x20000000 0x12345678`                                                                                     |
| 设断点       | `(gdb) break main`                                                                                              |
| 单步         | `(gdb) next` / `(gdb) step`                                                                                     |
| 复位暂停     | `reset halt`                                                                                                    |
| 查看Flash    | `flash probe 0` → `flash info 0`                                                                                |
| 退出         | Telnet: `shutdown`，GDB: `quit`                                                                                 |
