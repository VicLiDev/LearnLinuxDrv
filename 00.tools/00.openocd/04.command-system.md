## 4. 命令系统

### 4.1 架构概述

OpenOCD 的命令系统建立在 **Jim Tcl 解释器** 之上，实现了 C 语言函数与 TCL 脚本的无缝集成：

```
用户输入 (TCL脚本/命令行/telnet/gdb)
    ↓
Jim Tcl 解释器 (jimtcl/)
    ↓
jim_command_dispatch()     ← C 命令桥接函数
    ↓
COMMAND_HANDLER()         ← C 语言命令处理函数
```

### 4.2 核心数据结构

**命令注册表** (`src/helper/command.h:238`)：
```c
struct command_registration {
    const char *name;             // 命令名称
    command_handler_t handler;    // 处理函数 (可为 NULL，仅作为父命令)
    enum command_mode mode;       // 允许的执行模式
    const char *help;             // 帮助文本
    const char *usage;            // 用法说明
    const struct command_registration *chain;  // 子命令链
};
```

**命令上下文** (`src/helper/command.h:52`)：
```c
struct command_context {
    Jim_Interp *interp;           // Jim TCL 解释器
    enum command_mode mode;       // 当前模式
    struct target *current_target;  // 当前目标
    command_output_handler_t output_handler;  // 输出回调
    struct list_head *help_list;  // 帮助列表
};
```

**命令调用** (`src/helper/command.h:76`)：
```c
struct command_invocation {
    struct command_context *ctx;   // 命令上下文
    struct command *current;       // 当前命令
    const char *name;              // 命令名
    unsigned int argc;             // 参数数量
    const char **argv;             // 参数列表
    Jim_Obj * const *jimtcl_argv;  // TCL 原始参数
    char *output;                  // 输出缓冲区
};
```

### 4.3 命令定义方式

**使用宏定义命令处理函数** (`src/helper/command.h:132`)：
```c
#define COMMAND_HANDLER(name) static __COMMAND_HANDLER(name)
// 展开后为:
// static int name(struct command_invocation *cmd)
```

**命令定义示例**（取自实际代码）：
```c
// 定义命令处理函数
COMMAND_HANDLER(handle_version_command)
{
    if (CMD_ARGC != 0)
        return ERROR_COMMAND_SYNTAX_ERROR;
    command_print(CMD, OPENOCD_VERSION);
    return ERROR_OK;
}

// 注册命令
static const struct command_registration openocd_command_handlers[] = {
    {
        .name = "version",
        .handler = handle_version_command,
        .mode = COMMAND_ANY,
        .help = "show program version",
    },
    {
        .name = "init",
        .handler = handle_init_command,
        .mode = COMMAND_ANY,
        .help = " Initializes target and debug setup",
    },
    COMMAND_REGISTRATION_DONE  // 数组结束标记
};
```

### 4.4 命令系统常用宏

```c
CMD            // → cmd (当前 command_invocation)
CMD_CTX        // → cmd->ctx (命令上下文)
CMD_ARGC       // → cmd->argc (参数数量)
CMD_ARGV       // → cmd->argv (参数数组)
CMD_NAME       // → cmd->name (命令名)
CMD_DATA       // → cmd->current->jim_handler_data (命令私有数据)

// 参数解析
COMMAND_PARSE_NUMBER(u32, str, var)     // 解析数字参数
COMMAND_PARSE_ADDRESS(str, var)          // 解析地址参数
COMMAND_PARSE_BOOL(str, var, "on", "off") // 解析布尔参数

// 输出
command_print(CMD, "value: %d", val);        // 打印一行
command_print_sameline(CMD, "progress: %d", val); // 不换行打印
```

### 4.5 命令注册流程

```
1. 定义 COMMAND_HANDLER 函数
2. 填充 command_registration 结构体数组
3. 在模块初始化函数中调用 register_commands()
4. 所有模块的初始化函数通过 command_registrants[] 数组注册
5. setup_command_handler() 在启动时遍历该数组执行注册
```

### 4.6 子命令与命令链

命令通过 `chain` 字段支持层级结构：

```c
static const struct command_registration foo_sub_handlers[] = {
    {
        .name = "bar",
        .handler = handle_foo_bar,
        .mode = COMMAND_EXEC,
        .help = "do bar",
    },
    COMMAND_REGISTRATION_DONE
};

static const struct command_registration foo_handlers[] = {
    {
        .name = "foo",
        .mode = COMMAND_EXEC,
        .help = "foo commands",
        .chain = foo_sub_handlers,  // 子命令链
    },
    COMMAND_REGISTRATION_DONE
};
// 生成命令: foo bar
```
