## 5. JTAG/SWD 调试传输层

### 5.1 JTAG 核心概念

**JTAG (Joint Test Action Group)** 使用 4 根信号线通过有限状态机进行通信：

```
TCK - 测试时钟
TMS - 测试模式选择
TDI - 测试数据输入 (→目标)
TDO - 测试数据输出 (→主机)
可选: TRST - 测试复位, SRST - 系统复位
```

**TAP (Test Access Port)** 是 JTAG 链上的一个节点。多个 TAP 通过菊花链串联：

```
TDI → [TAP 1] → [TAP 2] → [TAP 3] → TDO
```

### 5.2 TAP 状态机

JTAG TAP 有 16 个状态（`src/jtag/jtag.h:37`）：

```
                ┌─── TMS=1 ───→ RESET
                │                   │
                │           TMS=0   │
                │    ┌──────────────┘
                │    │
                ↓    │
           IDLE ←────┘ TMS=0
           │    ↑
     TMS=1 │    │ TMS=0
           ↓    │
         SELECT-DR-SCAN ──→ CAPTURE-DR ──→ SHIFT-DR ──→ EXIT1-DR ──→ PAUSE-DR
                                                                 │
                                                          TMS=1  │
                                                                 ↓
                                                            UPDATE-DR
         SELECT-IR-SCAN ──→ CAPTURE-IR ──→ SHIFT-IR ──→ EXIT1-IR ──→ PAUSE-IR
                                                                 │
                                                          TMS=1  │
                                                                 ↓
                                                            UPDATE-IR
```

### 5.3 核心数据结构

**jtag_tap 结构** (`src/jtag/jtag.h`)：
```c
struct jtag_tap {
    char *chip;                    // 芯片名
    char *tapname;                 // TAP 名
    char *dotted_name;             // 层级名 "chip.tap"
    unsigned int abs_chain_position; // 在扫描链中的位置
    bool enabled;                  // 是否启用

    unsigned int ir_length;        // 指令寄存器长度
    uint32_t ir_capture_value;     // IR 捕获值
    uint8_t *expected;             // 期望的 IDCODE

    uint32_t idcode;              // 设备 ID 代码
    uint32_t *expected_ids;       // 期望的 ID 数组

    uint8_t *cur_instr;           // 当前指令
    bool bypass;                  // 是否处于旁路模式

    struct jtag_tap *next_tap;    // 链中的下一个 TAP
    void *priv;                   // 私有数据
};
```

### 5.4 SWD 协议

SWD (Serial Wire Debug) 是 ARM 专用的 2 线调试协议，比 JTAG 更高效：

| 特性     | JTAG                   | SWD                          |
|----------|------------------------|------------------------------|
| 信号线   | 4 线 (TCK/TMS/TDI/TDO) | 2 线 (SWCLK/SWDIO)           |
| 协议模型 | 状态机驱动             | 包格式 (packet-based)        |
| 设备支持 | 菊花链多设备           | 单个 DAP (Debug Access Port) |
| 典型速度 | 较低                   | 更高带宽                     |

**SWD 命令包** (`src/jtag/swd.h`)：
```c
#define SWD_CMD_START    (1 << 0)  // 起始位 (固定为 1)
#define SWD_CMD_APNDP    (1 << 1)  // 0=DP访问, 1=AP访问
#define SWD_CMD_RNW      (1 << 2)  // 0=写, 1=读
#define SWD_CMD_A32      (3 << 3)  // 地址位 [3:2]
#define SWD_CMD_PARITY   (1 << 5)  // 奇偶校验位
#define SWD_CMD_STOP     (0 << 6)  // 停止位 (固定为 0)
#define SWD_CMD_PARK     (1 << 7)  // 驱动为高
```

**SWD 驱动接口** (`src/jtag/swd.h`)：
```c
struct swd_driver {
    int (*init)(void);             // 初始化 SWD 链路
    int (*switch_seq)(enum swd_special_seq seq); // 特殊序列切换
    void (*read_reg)(uint8_t cmd, uint32_t *value, uint32_t ap_delay_hint);
    void (*write_reg)(uint8_t cmd, uint32_t value, uint32_t ap_delay_hint);
    int (*run)(void);              // 执行排队的事务
};
```

### 5.5 适配器与驱动

**适配器驱动接口** (`src/jtag/interface.h`)：
```c
struct adapter_driver {
    const char *name;                // 驱动名称
    unsigned int transport_ids;      // 支持的传输协议

    int (*init)(void);               // 初始化
    int (*quit)(void);               // 退出
    int (*reset)(int srst, int trst); // 复位

    int (*speed)(int speed);          // 设置速度
    int (*khz)(int khz, int *jtag_speed);  // kHz 转换
    int (*speed_div)(int speed, int *khz);  // 速度转 kHz

    struct jtag_interface *jtag_ops;   // JTAG 操作
    const struct swd_driver *swd_ops;   // SWD 操作
};
```

**驱动示例模式** (`src/jtag/drivers/` 下的每个驱动)：
```c
const struct adapter_driver xxx_adapter_driver = {
    .name = "xxx",
    .transport_ids = TRANSPORT_JTAG | TRANSPORT_SWD,
    .commands = xxx_command_handlers,

    .init = xxx_init,
    .quit = xxx_quit,
    .reset = xxx_reset,
    .speed = xxx_speed,
    .khz = xxx_khz,
    .speed_div = xxx_speed_div,

    .jtag_ops = &xxx_jtag_ops,
    .swd_ops = &xxx_swd_ops,
};
```

### 5.6 传输协议抽象

`src/transport/transport.c` 提供统一的传输协议抽象：

```c
#define TRANSPORT_JTAG           BIT(0)
#define TRANSPORT_SWD            BIT(1)
#define TRANSPORT_HLA_JTAG       BIT(2)  // 高层抽象 JTAG
#define TRANSPORT_HLA_SWD        BIT(3)  // 高层抽象 SWD
#define TRANSPORT_DAPDIRECT_JTAG BIT(4)
#define TRANSPORT_DAPDIRECT_SWD  BIT(5)
#define TRANSPORT_SWIM           BIT(6)  // STM8

struct transport {
    unsigned int id;
    int (*select)(struct command_context *ctx);
    int (*init)(struct command_context *ctx);
    struct list_head lh;
};
```
