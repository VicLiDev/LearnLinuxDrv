## 10. 架构设计总结

### 10.1 核心设计模式

**1. 面向对象的 C 语言实现**
OpenOCD 大量使用"结构体 + 函数指针"实现多态，等效于 C++ 的虚函数表：

```
target_type   → 相当于"基类"接口
cortexm_target → "派生类"实现
riscv_target   → "派生类"实现
```

**2. 插件式驱动注册**
所有驱动和目标类型通过静态数组或链表注册，启动时统一初始化：

```
command_registrants[] → 所有命令注册函数
target_types[]        → 所有目标类型
flash_drivers[]       → 所有 Flash 驱动
adapter_drivers[]     → 所有适配器驱动
```

**3. 分层抽象**
每一层通过接口与上下层交互，各层可独立替换：

```
用户接口层  ← 服务抽象 (service_driver)
命令层     ← 命令抽象 (command_registration)
功能层     ← 目标抽象 (target_type), Flash 抽象 (flash_driver)
传输层     ← 传输抽象 (transport)
硬件层     ← 适配器抽象 (adapter_driver)
```

**4. 事件驱动架构**
主循环使用 `select()` 多路复用，配合事件回调和定时器：

```
server_loop() {
    select()  → 等待 I/O 事件
    处理连接  → 路由到对应服务
    超时处理  → 执行定时器回调
}
```
